# Open Questions

## Phase 1 Structure

- Is the opening built around one obvious life-support restoration problem, or a small set of opening configurations?
- How much of the first stabilisation sequence should vary between runs?
- How much ship repair should happen before broader construction choices open up?
- How many viable early extension paths should exist to avoid linear solutions?

## Start Conditions

- What exact Phase 1 starting loadout values should be used for:
  - Finite oxygen reserve
  - Finite water reserve
  - Ration-based food
  - Limited power access
- Which systems begin as reserves only, and which begin as damaged infrastructure?
- What minimum pre-unlocked tech is needed so the player can act immediately without waiting?
- What minimum systems must be immediately buildable to avoid early dead time?
- How much starting material is required for repair versus extension without allowing full stabilisation?
- What depletion tempo best supports tension without early-run randomness spikes?

## Ship Layout and Structure

- Should the target test size begin at 64x64, 48x48, or another envelope?
- What is the target ship interior size envelope for fun versus performance?
- How much of the ship should be initially usable versus visually present but blocked?
- What fixed structural elements are mandatory for vessel identity?
- What percentage of structure should remain destructible for player agency?
- How much internal gating is useful before it feels like forced puzzle content?
- How should layout enforce constraint without blocking essential system placement?
- Should the optional hazard room be fixed content or vary by run?

## Intro Replayability

- What needs to vary so the intro is not rote, while still staying legible?
- Which elements should remain constant to preserve clarity of the core problem?
- How much randomness is required so repeat runs do not feel scripted?
- Which elements should vary run-to-run:
  - Resource distribution
  - Damage topology
  - Optional room hazard-reward type
- What is the acceptable length of the initial mandatory stabilisation burst?
- What is the maximum acceptable length for the opening before repetition risk rises?

## Descent Gate

- What exact hard capability gate unlocks descent?
- Which soft readiness factors modify post-descent survivability?
- How should the hard capability gate be represented in-world so it feels like ship restoration rather than arbitrary checklist progress?
- How should readiness be surfaced so players can make informed commit decisions?

## Mid-Game Handoff

- What exact support relationship should exist between ship and foothold after descent?
- How long should the ship remain relevant after descent?
- What support roles should remain available from the ship?
- How severe should initial foothold hostility be relative to late-game pressure?

## Hazard Roadmap

- Heat is the first concrete hazard path, but:
  - Should future hazard variants be selectable, randomized, or scenario-defined?
  - How many variants are realistic without diluting tuning quality?
  - What shared framework is needed before multi-hazard expansion is safe?
- Multi-hazard support is intentionally future-facing and not part of first implementation unless the heat-first path proves strong enough.

## Win Condition

- What victory form best proves sustained hostile-world mastery without collapsing into normal endgame escape?
- What final objective best represents mastery under sustained hostility?
- Should victory depend on continuous throughput over time, milestone completion, or both?
- How should sustained production or system-level success be evaluated clearly in gameplay?
- How should the design avoid a flat stockpile objective while keeping win-state evaluation clear?

## Prototype Questions

- Does a large damaged ship or ship-like habitat start actually feel good to play?
- Does the opening remain tense without becoming repetitive?
- Does the hard capability gate plus soft readiness layer create real judgment or just a disguised checklist?
- Does the opening create meaningful decisions or just a fixed sequence of actions?
